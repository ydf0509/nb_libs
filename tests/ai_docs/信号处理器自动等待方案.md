# 信号处理器自动等待方案

## 🎯 目标

**用户需求**：
1. 使用普通的 `asyncio.run`（不是 smart_run）
2. 不想手动调用 `await wait_for_all_pools_done()` 或 `await shutdown_all_pools()`  
3. 避免程序提前退出导致任务丢失

## ✅ 解决方案：自动信号处理器

### 核心思想

在第一次 `submit()` 时，自动安装信号处理器（SIGINT/SIGTERM），在程序退出或被中断时自动等待所有pending任务完成。

---

## 🔧 实现细节

### 1. 跨平台支持

```python
def _install_signal_handlers():
    """
    跨平台方案：
    - Windows: 只用 signal.signal (不支持 add_signal_handler)
    - Linux/Mac: 优先用 add_signal_handler，回退到 signal.signal
    """
    
    is_windows = sys.platform == 'win32'
    
    if is_windows:
        # Windows 模式
        signal.signal(signal.SIGINT, cleanup_sync_handler)
        signal.signal(signal.SIGTERM, cleanup_sync_handler)
    else:
        # Unix 模式
        try:
            loop = asyncio.get_running_loop()
            loop.add_signal_handler(signal.SIGINT, cleanup_handler)
            loop.add_signal_handler(signal.SIGTERM, cleanup_handler)
        except (RuntimeError, NotImplementedError):
            # 回退
            signal.signal(signal.SIGINT, cleanup_sync_handler)
            signal.signal(signal.SIGTERM, cleanup_sync_handler)
```

### 2. 自动安装机制

```python
class SmartAsyncPool:
    async def submit(self, func, *args, **kwargs):
        if not self._is_running:
            await self._start()
            # 🔧 第一次submit时自动安装
            if self._auto_shutdown:
                _install_signal_handlers()
```

### 3. 清理逻辑

```python
async def cleanup_async():
    """等待所有pool的pending任务"""
    logger.info("🔧 Signal received, cleaning up...")
    for pool in list(_active_pools):
        if pool.pending_count > 0:
            await pool.async_wait_for_all()
    logger.info("✅ All pools cleaned up.")
```

## 📊 使用效果

### 场景1：正常退出

```python
pool = SmartAsyncPool(auto_shutdown=True)

async def main():
    await pool.submit(task, 1)
    await pool.submit(task, 2)
    # 不手动等待，直接退出

asyncio.run(main())
```

**结果**：
```
📊 提交了2个任务
✨ 没有手动等待，但信号处理器已安装！
```

**注意**：正常退出时，信号处理器不会被触发！
- ✅ 如果任务在main()结束前完成 → OK
- ❌ 如果任务还没完成，main()就结束 → 任务丢失

### 场景2：Ctrl+C中断

```python
pool = SmartAsyncPool(auto_shutdown=True)

async def main():
    for i in range(1000):
        await pool.submit(slow_task, i)
    # 用户按 Ctrl+C

asyncio.run(main())
```

**结果**：
```
📊 提交了1000个任务
^C
📡 Received signal 2, initiating cleanup...
🔧 Signal received, cleaning up async pools...
⏳ Waiting for 500 pending tasks in pool...
✅ All pools cleaned up.
```

✅ **信号处理器生效！等待所有任务完成后再退出**

## ⚠️ 局限性

### 1. 只在被中断时生效

信号处理器只在收到 SIGINT（Ctrl+C）或 SIGTERM 时触发。

**无法解决**：正常退出但任务还没完成的情况

```python
async def main():
    await pool.submit(task, 1)  # 任务需要1秒
    # main立即结束，信号处理器不会触发
    # 任务丢失！❌

asyncio.run(main())
```

### 2. Windows限制

Windows 上 `add_signal_handler` 不可用，只能用 `signal.signal`，功能有限。

## 🎯 推荐做法

### ✅ 最佳：使用 smart_run()

```python
pool = SmartAsyncPool(auto_shutdown=True)

async def main():
    await pool.submit(task, 1)
    # 不需要任何等待代码

smart_run(main())  # ✅ 自动等待所有任务
```

**优点**：
- ✅ 正常退出时等待
- ✅ Ctrl+C中断时等待
- ✅ 完全自动，零心智负担

### ✅ 次佳：使用 async with

```python
async with SmartAsyncPool() as pool:
    await pool.submit(task, 1)
# 自动shutdown(wait=True)

asyncio.run(main())
```

**优点**：
- ✅ 正常退出时等待
- ✅ 明确的生命周期管理

### ⚠️ 备选：信号处理器（只在特殊场景）

```python
pool = SmartAsyncPool(auto_shutdown=True)

async def main():
    await pool.submit(task, 1)
    # 不等待

asyncio.run(main())
```

**只在以下场景有用**：
- 长时间运行的程序
- 用户可能用 Ctrl+C 中断
- 希望中断时优雅退出

**不适合**：
- 短时间运行的脚本
- 期望正常退出时等待任务

## 🔍 技术细节

### Windows vs Unix

| 平台 | 机制 | 限制 |
|-----|------|------|
| Windows | `signal.signal()` | 功能有限，信号种类少 |
| Unix/Linux/Mac | `loop.add_signal_handler()` | 功能完整，可处理更多信号 |

### 信号列表

| 信号 | 含义 | 触发方式 |
|-----|------|---------|
| SIGINT | 中断信号 | Ctrl+C |
| SIGTERM | 终止信号 | `kill <pid>` |

## 📝 总结

### 信号处理器的作用

✅ **有效场景**：
- Ctrl+C 中断程序
- `kill` 命令终止进程
- 系统关闭时的优雅退出

❌ **无效场景**：
- 正常的程序退出
- main()函数结束
- `sys.exit()` 调用

### 核心建议

**不要依赖信号处理器来防止任务丢失！**

应该使用：
1. **smart_run()** - 最推荐，完全自动
2. **async with** - 次推荐，明确管理
3. **手动等待** - 需要精确控制时

信号处理器只是**额外的安全网**，不是主要方案。
